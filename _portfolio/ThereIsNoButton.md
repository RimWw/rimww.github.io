This is a UGC project made in [Genshin Impact's Miliastra Wonderland](https://genshin.hoyoverse.com/en/news/detail/159060). In this project, I learned to use the available framework of Miliastra to create a short 3-minute linear adventure game (without narrative) about finding buttons in a stage that tries to prevent the player from finding buttons in various ways. Inspired by games such as [There Is No Game: Wrong Dimension](https://store.steampowered.com/app/1240210/There_Is_No_Game_Wrong_Dimension/) and [Do Not Press The Button (Or You'll Delete The Multiverse)](https://store.steampowered.com/app/2336120/Do_Not_Press_The_Button_Or_Youll_Delete_The_Multiverse/).

Here's a page of the stage (Unofficial): [Miliastra.games page](https://miliastra.games/games/there-is-no-button-7573645601-en)

Miliastra is Genshin Impact's User Generated Content (UGC) framework and platform that let us create stages with their built in tools, framework, and assets and publish it to be freely played by other players. Curious, I tried learning it and made this stage to implement what I learned about this framework. At the time, custom text was a locked feature that require creators to first publish a stage and get some plays before unlocking it. With that, I chose to make a stage that can be played without custom text, having the gameplay intuitive enough that it can be easily played: Pressing a button to open a door.

With that idea, I tried to think of how to make a game around pressing buttons as interesting as possible while still being intuitive. I thought about trying to subvert player's expectation about a simple mechanic and turn it into not so simple mechanic, like hiding it. Soon, I remembered that there are games out there about subverting player expectation and perception, messing with the player in silly ways, often in humorous way, and some with meta-narrative. I'm talking about games like [The Stanley Parable](https://store.steampowered.com/app/221910/The_Stanley_Parable/), [Superliminal](https://store.steampowered.com/app/1049410/Superliminal/), the inspirations listed above, and many more.

With that I embraced the intent of "messing" with players and designed this stage with that in mind, referencing some part of the games I listed above that inspired me... several times. In general, in this stage I learned and tested my knowledge of the framework such as:

- Manipulating object behaviour and appearance through script
- Moving objects with motion device component
- Communicating game events, states, and actions through script
- Use of SFX and VFX
- and many more...

The use of the Miliastra framework eliminated the need to program many base game functions, sourcing high quality assets, and allowed more focus on game design, level design and top-level programming, albeit at the cost of reduced customization, flexibility, and imposed performance limits.

This stage gained a very positive ratings of **~98%** from players, with many praising the creative design and asked for more levels/content update. Although the stage is rather simple, I learned a lot about the framework from this project, and will continue to learn more to try make stages that are fun and immersive.
